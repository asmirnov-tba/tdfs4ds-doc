    Datasets — build_dataset¶

    A dataset is a denormalised Teradata view that joins selected feature versions for a given entity. It is always a view, never a materialised table.

    Resolve feature versions first¶

    from tdfs4ds.feature_store.feature_query_retrieval import get_feature_versions

selected_features = get_feature_versions(
    entity_name = 'CUSTOMER_ID',
    features    = ['nb_transactions', 'total_amount', 'avg_amount'],
)

    selected_features is {feature_name: process_uuid}. Always go through get_feature_versions — manual construction skips type validation and is brittle to renames.

    Build¶

    dataset = tdfs4ds.build_dataset(
    entity_id         = {'CUSTOMER_ID': 'BIGINT'},
    selected_features = selected_features,
    view_name         = 'DS_CUSTOMER_SPEND',
    comment           = 'Customer spending features',
)

dataset.shape

    Query strategy — grouped parameter¶

    By default build_dataset emits one sub-query and one JOIN per feature:

    SELECT A1.CUSTOMER_ID, A1.FEATURE_VALUE AS nb_transactions,
       A2.FEATURE_VALUE AS total_amount, ...
FROM (SEQUENCED VALIDTIME SELECT ... WHERE FEATURE_ID=1) A1
JOIN (SEQUENCED VALIDTIME SELECT ... WHERE FEATURE_ID=2) A2
  ON A1.CUSTOMER_ID = A2.CUSTOMER_ID
...

    Pass grouped=True to activate the pivot strategy instead — features that share the same source table and process ID are collapsed into a single MAX(CASE WHEN FEATURE_ID=… THEN FEATURE_VALUE END) … GROUP BY sub-query:

    dataset = tdfs4ds.build_dataset(
    entity_id         = {'CUSTOMER_ID': 'BIGINT'},
    selected_features = selected_features,
    view_name         = 'DS_CUSTOMER_SPEND',
    comment           = 'Customer spending features — grouped strategy',
    grouped           = True,
)

    The generated SQL looks like:

    SELECT A1.CUSTOMER_ID
  ,MAX(CASE WHEN FEATURE_ID=1 THEN FEATURE_VALUE END) AS nb_transactions
  ,MAX(CASE WHEN FEATURE_ID=2 THEN FEATURE_VALUE END) AS total_amount
  ...
FROM (SEQUENCED VALIDTIME SELECT ...
      WHERE FEATURE_VERSION='...' AND FEATURE_ID IN (1, 2, ...)) A1
GROUP BY A1.CUSTOMER_ID
HAVING MAX(CASE WHEN FEATURE_ID=1 THEN FEATURE_VALUE END) IS NOT NULL
   AND MAX(CASE WHEN FEATURE_ID=2 THEN FEATURE_VALUE END) IS NOT NULL

    Single-feature groups are unaffected

    When a group contains only one feature, build_dataset emits a plain FEATURE_ID = N scan with no GROUP BY — no performance penalty.

    When to use grouped=True

    The Teradata optimizer normalises both patterns to equivalent execution plans for typical queries. grouped=True is most beneficial when you have 20+ features from the same table/process and want to reduce the structural JOIN count — useful if the optimizer's rewrite budget is exhausted by very wide feature sets.

    Inspect the generated SQL¶

    Use return_query=True to retrieve the DDL without executing it or touching the dataset catalog:

    sql = tdfs4ds.build_dataset(
    entity_id         = {'CUSTOMER_ID': 'BIGINT'},
    selected_features = selected_features,
    view_name         = 'DS_CUSTOMER_SPEND',
    return_query      = True,
)
print(sql)

    Point-in-time snapshots¶

    tdfs4ds.FEATURE_STORE_TIME = '2024-01-01 00:00:00'

dataset_jan = tdfs4ds.build_dataset(
    entity_id         = {'CUSTOMER_ID': 'BIGINT'},
    selected_features = selected_features,
    view_name         = 'DS_CUSTOMER_SPEND_2024_01_01',
)

tdfs4ds.FEATURE_STORE_TIME = None

    The dataset view embeds a VALIDTIME AS OF TIMESTAMP '2024-01-01 00:00:00' qualifier inside each sub-query, so it always returns values as of that point even when queried later.

    Both grouped=False (default) and grouped=True work with time travel. The temporal qualifier applied inside each sub-query changes automatically:

    Mode Sub-query temporal qualifier
    Current (FEATURE_STORE_TIME = None) SEQUENCED VALIDTIME
    Time travel (FEATURE_STORE_TIME = '…') VALIDTIME AS OF TIMESTAMP '…' 
    tdfs4ds.FEATURE_STORE_TIME = '2024-01-01 00:00:00'

dataset_grouped_past = tdfs4ds.build_dataset(
    entity_id         = {'CUSTOMER_ID': 'BIGINT'},
    selected_features = selected_features,
    view_name         = 'DS_CUSTOMER_SPEND_2024_01_01_GROUPED',
    grouped           = True,
    feature_store_time = True,
)

tdfs4ds.FEATURE_STORE_TIME = None

    Augment a source DataFrame in place¶

    augment_source_with_features is useful for ad-hoc model exploration when you do not want to register a permanent dataset view:

    from tdfs4ds.dataset.builder import augment_source_with_features

augmented = augment_source_with_features(
    source_df         = my_source_df,
    entity_id         = {'CUSTOMER_ID': 'BIGINT'},
    selected_features = selected_features,
)

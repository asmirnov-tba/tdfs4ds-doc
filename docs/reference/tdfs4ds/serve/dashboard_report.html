    tdfs4ds.serve.dashboard_report ¶

    tdfs4ds.serve.dashboard_report¶

    Render a compelling, self-contained interactive HTML report from a plain data dict describing the feature store (inventory, insight facts, technical storage-table sizes, orphan features, process run-history, freshness alerts, lineage dependency counts).

    Same philosophy as :mod:tdfs4ds.benchmark.report: the HTML is fully offline — inline SVG charts + scoped CSS, no JavaScript — so it renders identically in any browser, prints cleanly, embeds directly in the admin panel, and exports to a single .html file. Interactivity is via native <details> expand/collapse.

    The builder consumes only JSON-serialisable data (records / numbers), never Plotly figures, so the same document shows on-screen and downloads byte-identical.

    Public: :func:build_dashboard_report, :func:write_dashboard_report.

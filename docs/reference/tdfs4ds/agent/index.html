    skill_generate_candidates ¶

    skill_generate_candidates(sql_query: str, user_request: str, original_analysis: dict, ddls: dict, lineage_info: dict, history: Optional[list]) -> tuple[list[dict], list[dict], Optional['StrategiesOutput']]

    Strategy-based candidate generation with multi-round refinement.

    Pipeline¶
    1. Plan strategies — combinatorial: one strategy per [You] item (sequential), then one per pair (simultaneous), then triplets, etc. No LLM call needed; capped at QUERY_OPTIMIZER_MAX_CANDIDATES.
    2. Execute each strategy — For each strategy sequentially: a. Generate one candidate implementing that strategy. b. EXPLAIN the candidate → extract remaining [You] items. c. If [You] items remain, refine the candidate (up to QUERY_OPTIMIZER_MAX_REFINE_ROUNDS rounds), each round targeting the [You] items from the previous round. d. Early stop the entire pipeline when a strategy achieves user_score = 5/5 (no [You] items remaining).
    Returns¶

    candidate_analyses : list[dict] One entry per strategy executed. Each has index, sql, strategy, strategy_description, strategy_targets, rationale, analysis, and rounds (list of per-round dicts). steps : list[dict] Pipeline step records for skill_optimize_query. strategy_plan : StrategiesOutput | None The raw plan returned by the strategy-planning LLM call (None when the query was already optimal and no strategies were generated).

    skill_simplify_query ¶

    skill_simplify_query(sql_query: Optional[str] = None, process_id: Optional[str] = None, max_fix_attempts: int = 3) -> dict

    Simplify a Teradata SQL query by removing unnecessary nesting.

    Applies deterministic passes (comment stripping, identity-wrapper flattening, reserved-word alias fixes) followed by an LLM pass that merges derived-table layers into CTEs and inlines date arithmetic.

    The simplified SQL is validated with EXPLAIN; if the score regresses below the original, the deterministic-only version is returned.

    Can be used standalone or as a pre-step in skill_optimize_query.

    		 None
    max_fix_attempts int

    Max syntax-error repair retries for the LLM output.

    		 3

    Returns:

    Type Description

    dict with keys simplified_sql, original_sql, simplified,
    dict

    Parameters:

    		 None

    Returns:

    Type Description
    Optional[str]

    Matching DATA_DOMAIN string, or None if not found.

    Cross-store federation limit (v0.3.1.148): the catalog scanned here is the federated UNION across the primary + read-only stores, but resolution returns a bare DATA_DOMAIN with NO store tag (and _with_domain only switches tdfs4ds.DATA_DOMAIN, not the active store).     launch_chatbot_app ¶

    launch_chatbot_app(port: int = 7860, server_name: str = '0.0.0.0', title: str = 'Feature Store Assistant', description: str = None, inline: bool = False, share: bool = False, thread_id: Optional[str] = None) -> gr.Blocks

    Launch the consumer-agent chatbot with a Teradata-styled tabbed interface.

    The layout uses a left-oriented tab nav (sidebar look) + a hero header, themed via :data:_TD_CSS (Teradata orange/navy palette). Tabs:

    • 💬 Chatbot — the conversational feature-store assistant (left) plus a right Additional skills pane that lists the plugin skills in TDFS4DS_SKILLS_FOLDER and can import/export a skill's SKILL.md.
    • 🔗 Lineage — interactive pyvis dependency graph of the full persisted lineage (FS_LINEAGE_GRAPH). Click Refresh lineage graph to load.
    • 🧪 Evaluation — generate / import / run evaluation scenarios + report.

    • ⚙️ Parameters — live admin controls:

    • Feature store database — switch the active feature store. The target is validated with tdfs4ds.connect(database=..., create_if_missing=False); a database that does not contain a feature store is rejected (no switch).

    • Vector store — backend (chroma/teradata) + Chroma path, and a Refresh/Rebuild button (build_vector_index(force_rebuild=True)).
    • Instruct model / Embedding model — provider, model, URL, API key, proxy (and embedding dimension), applied to the tdfs4ds globals.

    Applying any change that affects the data or the model resets the agent (reset_consumer_agent()), rebuilds the index when needed, rotates the conversation thread, and clears the chat — so the session starts clean.

    The existing :func:launch_chatbot / :func:launch_chatbot_with_index are unchanged; this is an additive wrapper.

    Returns:

    Type Description
    Blocks

    The Gradio Blocks demo instance.

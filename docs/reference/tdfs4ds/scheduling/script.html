    tdfs4ds.scheduling.script¶

    tdfs4ds.scheduling.script ¶

    Generate a standalone Python script that executes a parallel-execution plan (v0.3.1.147).

    The plan produced by :func:tdfs4ds.analyze_parallel_plan is turned into a self-contained .py that runs the closure's processes wave by wave: sequential across waves (dependency order), parallel within a wave. Parallelism uses concurrent.futures.ProcessPoolExecutor — one OS process per unit — NOT threads, because tdfs4ds.run and ingestion mutate module-level globals (DATA_DOMAIN / FEATURE_STORE_TIME / RUN_ID / PROCESS_TYPE) and use teradataml's shared SQLAlchemy pool, which corrupts under thread fan-out. Each worker process bootstraps its own Teradata context from the TDFS4DS_TD_* environment, so its global mutations never collide.

    generate_execution_script returns the script as a string (inspect without executing — the repo's return_query idiom); write_execution_script saves it under tdfs4ds.SCHEDULING_SCRIPT_DIR.

    _TEMPLATE module-attribute ¶

    _TEMPLATE = '#!/usr/bin/env python\n"""\nAuto-generated by tdfs4ds {version} on {generated_at}.\n\nParallel-execution script: {n_processes} process(es) across {n_waves} wave(s).\nWaves run sequentially (dependency order); processes within a wave run in\nparallel, one OS process each (safe re: tdfs4ds module globals + teradataml\npool). Each worker opens its own Teradata context from TDFS4DS_TD_* env.\n\nRun:   python {basename}\n"""\nimport json\nimport sys\nfrom concurrent.futures import ProcessPoolExecutor, as_completed\n\nPLAN = json.loads({plan_literal})\nFORCE_COMPUTE = {force_compute}\nMAX_WORKERS = {max_workers}   # None = one worker per parallel unit\nMODE = {mode!r}\n\n\ndef _bootstrap():\n    """Open this worker\'s own Teradata context + tdfs4ds config from env."""\n    import tdfs4ds\n    from tdfs4ds.utils.reconnect import reconnect_teradata\n    ok = reconnect_teradata()\n    if not ok:\n        raise RuntimeError(\n            "reconnect_teradata() failed - set TDFS4DS_TD_HOST/USERNAME/PASSWORD "\n            "(and optionally _LOGMECH/_DATABASE) in the environment."\n        )\n    if PLAN.get("schema"):\n        tdfs4ds.SCHEMA = PLAN["schema"]\n    if PLAN.get("schema_view"):\n        tdfs4ds.SCHEMA_VIEW = PLAN["schema_view"]\n    return tdfs4ds\n\n\ndef _execute_pid(pid):\n    """Run one process; drive its TimeManager via roll_out when it has one."""\n    tdfs4ds = _bootstrap()\n    meta = PLAN["processes"].get(pid, {{}})\n    dom = meta.get("data_domain")\n    if dom:\n        tdfs4ds.DATA_DOMAIN = dom\n    tm = meta.get("time")\n    if tm and tm.get("view"):\n        from tdfs4ds.utils.time_management import TimeManager\n        tmgr = TimeManager(table_name=tm["view"], schema_name=tm["database"])\n        tdfs4ds.roll_out([pid], tmgr, force_compute=FORCE_COMPUTE)\n    else:\n        tdfs4ds.run(pid, force_compute=FORCE_COMPUTE)\n    return pid\n\n\ndef _run_group(pids):\n    """Run a serialize-together group sequentially inside a single worker."""\n    for pid in pids:\n        print("  [proc] running", pid, flush=True)\n        _execute_pid(pid)\n    return pids\n\n\ndef _wave_units(wave_idx, wave):\n    """Group write-conflicting processes into one unit; others are singletons."""\n    conflict = {{}}\n    for grp in PLAN["serialize_together"]:\n        if grp.get("wave") == wave_idx:\n            members = [p for p in grp["processes"] if p in wave]\n            if len(members) > 1:\n                rep = members[0]\n                for p in members:\n                    conflict[p] = rep\n    units, seen = [], set()\n    for grp in PLAN["serialize_together"]:\n        if grp.get("wave") == wave_idx:\n            members = [p for p in grp["processes"] if p in wave and p not in seen]\n            if len(members) > 1:\n                units.append(members)\n                seen.update(members)\n    for pid in wave:\n        if pid not in seen:\n            units.append([pid])\n            seen.add(pid)\n    return units\n\n\ndef _run_sequential():\n    for k, wave in enumerate(PLAN["waves"]):\n        print("Wave {{}}: {{}} process(es) (sequential)".format(k, len(wave)), flush=True)\n        for pid in wave:\n            _execute_pid(pid)\n    print("All waves complete.", flush=True)\n\n\ndef _run_parallel():\n    for k, wave in enumerate(PLAN["waves"]):\n        units = _wave_units(k, wave)\n        print("Wave {{}}: {{}} parallel unit(s) / {{}} process(es)".format(k, len(units), len(wave)), flush=True)\n        workers = MAX_WORKERS or max(1, len(units))\n        with ProcessPoolExecutor(max_workers=workers) as ex:\n            futs = {{ex.submit(_run_group, u): u for u in units}}\n            for fut in as_completed(futs):\n                fut.result()   # re-raise worker failures\n    print("All waves complete.", flush=True)\n\n\ndef main():\n    print("tdfs4ds parallel execution | {n_waves} wave(s) | {n_processes} process(es)", flush=True)\n    if MODE == "sequential":\n        _run_sequential()\n    else:\n        _run_parallel()\n\n\nif __name__ == "__main__":\n    main()\n'

    generate_execution_script ¶

    generate_execution_script(plan: Dict[str, Any], *, out_path: Optional[str] = None, mode: str = 'process_pool', force_compute: bool = False, max_workers: Optional[int] = None) -> str

    Turn an :func:analyze_parallel_plan result into a runnable .py script.

    Parameters¶

    plan : dict The plan returned by :func:tdfs4ds.analyze_parallel_plan. out_path : str | None If given, also write the script there (see also :func:write_execution_script). mode : {'process_pool', 'sequential'} 'process_pool' (default) parallelises each wave across processes; 'sequential' runs everything in order (debugging). force_compute : bool Passed to run/roll_out in the generated worker. max_workers : int | None Cap on parallel workers per wave (None = one per unit).

    Returns¶

    str The script source.
